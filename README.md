# ChatInsight — Dialogue Summarization

## Project Summary

ChatInsight is a small project that provides dialogue summarization using a fine-tuned T5 model with a Streamlit-based UI. The app loads a locally saved model from `saved_summary_model/`, accepts conversation text via a text area, and returns a concise summary generated by the model.

Key components:
- `app.py` — Streamlit application that loads the model and exposes a simple UI for generating summaries.
- `summary.py` — Colab-style notebook script that demonstrates dataset preparation (Samsum), fine-tuning a `t5-small` model, and exporting a fine-tuned model to `fine_tuned_model` (which can be packaged as `saved_summary_model`).
- `saved_summary_model/` — Saved tokenizer & model configuration used by `app.py`.

## Technologies

- Python 3.8+ (tested with 3.8–3.11)
- Transformers (Hugging Face) — `T5Tokenizer`, `T5ForConditionalGeneration`
- PyTorch — model runtime
- Streamlit — lightweight web UI

## Quick Setup

1. (Recommended) Create and activate a virtual environment:

```bash
python -m venv .venv
source .venv/bin/activate
```

2. Install dependencies (basic):

```bash
pip install -U pip
pip install streamlit transformers torch sentencepiece
```

Note: Installing `torch` may require a platform-specific wheel (CUDA-enabled or CPU-only). If you need GPU support, follow the official PyTorch install instructions at https://pytorch.org/get-started/locally/.

3. Ensure `saved_summary_model/` exists at repository root and contains the saved model/tokenizer files (the repository already contains this folder with `config.json`, `tokenizer.json`, etc.).

## Run the App

From the repository root run:

```bash
streamlit run app.py
```

Open the URL Streamlit outputs (usually `http://localhost:8501`) and paste a dialogue into the text area, then click `Generate Summary`.

## How the Model Is Trained (from `summary.py`)

- `summary.py` contains a Colab-style fine-tuning workflow using the Samsum dataset: cleaning text, tokenization, building `Trainer` and `TrainingArguments`, training a `t5-small` model, and saving it to a local folder (`./fine_tuned_model`).
- After training, the example zips the folder as `saved_summary_model.zip` for download. The same saved model structure is what `app.py` expects at `./saved_summary_model`.

## Notes & Observations from Code

- `app.py`:
  - Uses `@st.cache_resource` to cache model load for Streamlit sessions.
  - Input trimming and simple regex cleaning are applied before tokenization.
  - Tokenization uses `max_length=512` and `padding="max_length"`; generation uses `num_beams=4` and `max_length=150`.

- `summary.py`:
  - Demonstrates sampling (smaller subsets for quicker fine-tuning in Colab) and a typical Trainer-based training loop.
  - Contains inline shell commands and notebook-specific constructs (e.g., `!pip install`)—these should be adapted for script-based runs.
